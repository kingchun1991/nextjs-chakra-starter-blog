# 國際化 (i18n) 指南

此部落格模板使用 [next-intl](https://next-intl-docs.vercel.app/) 內建可選的國際化 (i18n) 支援。**預設情況下，i18n 已停用**，以提供更簡單的單語言體驗。當您需要多語言支援時，可以輕鬆啟用。

## 快速概覽

**主要功能：**
- 🎯 **預設停用** - 開箱即用的簡單單語言設定
- 🌍 需要時可使用多語言支援
- 🔄 使用標題按鈕輕鬆切換語言（啟用時）
- 📝 基於語言環境的內容組織
- 🎯 SEO 友善的語言環境路由
- ⚡ 純英文部落格無需任何配置

## 啟用國際化

要啟用多語言支援：

### 1. 建立環境檔案

在專案根目錄建立 `.env.local` 檔案（您可以選擇從範例複製）：

```bash
cp .env.local.example .env.local
```

**注意：** `.env.local` 檔案是**完全可選的**。沒有它，模板預設為純英文模式。

### 2. 啟用 i18n

在 `.env.local` 中新增此行：

```bash
NEXT_PUBLIC_ENABLE_I18N=true
```

### 3. 重新啟動開發伺服器

啟用或停用 i18n 後，重新啟動您的開發伺服器：

```bash
pnpm dev
```

就是這樣！您的部落格現在支援英文和繁體中文。語言切換器將自動出現在標題中。

## 運作原理

模板使用 **Next.js 重寫（rewrites）** 在 i18n 停用時提供簡潔的 URL：

### 當 i18n 啟用時 (`NEXT_PUBLIC_ENABLE_I18N=true`)

1. **基於語言環境的路由**：所有路由都使用 `[locale]` 目錄結構（例如 `/en/blog` 或 `/zh-TW/blog`）
2. **內容組織**：部落格文章依語言環境組織在 `content/blog/[locale]/`
3. **UI 翻譯**：介面文字從 `messages/[locale].json` 載入
4. **語言切換器**：出現在標題中用於切換語言
5. **中介軟體**：使用 next-intl 處理語言環境偵測和路由

### 當 i18n 停用時（預設）

1. **簡潔網址**：路由不帶語言環境前綴（例如 `/blog` 而非 `/en/blog`）
2. **URL 重寫**：Next.js 在內部將 `/blog` 重寫為 `/en/blog`，對使用者不可見
3. **單一語言**：僅使用 `content/blog/en/` 的英文內容
4. **無語言切換器**：語言切換器隱藏
5. **簡化配置**：無語言環境路由開銷

**注意：** 切換 i18n 旗標後只需重新啟動開發伺服器即可，無需重新建置！

## 支援的語言

預設情況下，當 i18n 啟用時，此模板支援：

- **英文 (en)** - 預設語言
- **繁體中文 (zh-TW)**

您可以按照以下步驟輕鬆新增更多語言。

## 新增語言

要新增新語言（例如西班牙文）：

### 1. 更新路由配置

編輯 `src/i18n/routing.ts`：

```typescript
export const routing = defineRouting({
  locales: ['en', 'zh-TW', 'es'], // 新增 'es' 表示西班牙文
  defaultLocale: 'en',
  localePrefix: 'as-needed',
});
```

### 2. 建立翻譯檔案

建立新的訊息檔案 `messages/es.json`：

```json
{
  "header": {
    "home": "Inicio",
    "blog": "Blog",
    "tags": "Etiquetas",
    "about": "Acerca de"
  },
  "blog": {
    "readMore": "Leer más",
    "backToBlog": "Volver al blog"
  }
}
```

### 3. 建立內容目錄

為西班牙文部落格內容建立新目錄：

```bash
mkdir -p content/blog/es
```

### 4. 新增翻譯內容

將您的部落格文章複製到新目錄並翻譯它們：

```bash
cp content/blog/en/get-started.mdx content/blog/es/get-started.mdx
# 然後翻譯西班牙文版本的內容
```

### 5. 更新語言切換器（可選）

如果您想自訂語言切換器標籤，請編輯 `src/lib/layout/language-switcher.tsx`。

## 停用國際化

要停用 i18n 並只使用英文：

### 選項 1：使用環境變數（建議）

i18n 在以下情況下**預設停用**：
- 不存在 `.env.local` 檔案，或
- `NEXT_PUBLIC_ENABLE_I18N` 未定義，或
- 設定 `NEXT_PUBLIC_ENABLE_I18N=false`

只需移除 `.env.local` 檔案或移除/變更該變數即可停用 i18n。

### 選項 2：完全移除 i18n

如果您從不打算使用 i18n：

1. 解除安裝 next-intl：
```bash
pnpm remove next-intl
```

2. 移除 i18n 目錄：
```bash
rm -rf src/i18n messages content/blog/zh-TW
```

3. 更新程式碼中的匯入以移除 next-intl 參照

## 翻譯內容

### 部落格文章

每篇部落格文章都應該有每種支援語言的版本：

```
content/
└── blog/
    ├── en/
    │   ├── my-post.mdx
    │   └── another-post.mdx
    └── zh-TW/
        ├── my-post.mdx
        └── another-post.mdx
```

**重要：** 檔案名稱在各語言中應該相同（例如 `my-post.mdx`），但內部內容可以完全翻譯。

### UI 元素

在訊息檔案中翻譯介面文字：

**messages/en.json:**
```json
{
  "header": {
    "home": "Home",
    "blog": "Blog"
  }
}
```

**messages/zh-TW.json:**
```json
{
  "header": {
    "home": "首頁",
    "blog": "部落格"
  }
}
```

然後在元件中使用它們：

```tsx
import { useTranslations } from 'next-intl';

export function MyComponent() {
  const t = useTranslations('header');

  return <h1>{t('home')}</h1>;
}
```

## 語言切換器

當 i18n 啟用時，語言切換器會出現在標題中，允許使用者在可用語言之間切換。它：

- 🌐 顯示地球圖示
- 🔄 切換語言時保留目前頁面
- 📍 使用新的語言環境前綴更新 URL
- ✨ 具有 Chakra UI 樣式的平滑過渡
- 🎯 停用 i18n 時自動隱藏

您可以在 `src/lib/layout/language-switcher.tsx` 中自訂其外觀。

## 最佳實踐

1. **從簡單開始**：從停用 i18n（單語言）開始，需要時再啟用
2. **一致的結構**：在所有語言環境中保持相同的檔案結構
3. **SEO**：每種語言都有自己的 URL，對 SEO 很有幫助
4. **備援**：如果語言環境中缺少內容，模板會回退到英文
5. **測試**：始終測試語言切換以確保內容已正確翻譯
6. **輕鬆升級**：模板使用者可以升級到較新版本，而不會破壞 i18n 變更

## 技術細節

- **函式庫**：next-intl v4.8.3
- **路由**：使用動態 `[locale]` 區段的 App Router
- **中介軟體**：基於環境變數的條件式中介軟體
- **內容載入**：語言環境感知的 MDX 檔案載入
- **預設行為**：i18n 停用（僅英文）以保持簡單
- **配置**：單一環境變數（`NEXT_PUBLIC_ENABLE_I18N`）
